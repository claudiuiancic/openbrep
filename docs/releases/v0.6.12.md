# OpenBrep v0.6.12

Release type: GDL knowledge base calibration and Pro direction planning.

## Compatibility

- macOS package: Apple Silicon only (`arm64`, M1/M2/M3/M4).
- Minimum macOS: macOS 14 Sonoma or later.
- Intel Mac is not covered by the current `OpenBrep-free-macOS.zip` asset.
- The current macOS zip is not yet Developer ID signed and notarized. If
  Gatekeeper blocks the app, unzip it and run:
  `xattr -dr com.apple.quarantine /path/to/OpenBrep`, then start
  `OpenBrep.command` again. A signed and notarized package is planned.
- Windows package: built by GitHub Actions and provided as `OpenBrep-free-Windows.zip`; Windows runtime smoke still needs validation on a real Windows machine or VM.

## Why

OpenBrep should generate professional GDL library parts, not only simple
geometric blocks. This release closes the current GDL knowledge calibration
round so generation can rely on verified command semantics, HSF parameter
structure, 2D/3D expression rules, and archetype boundaries.

## What changed

- Knowledge verification plan:
  - Marked P0-P6 as completed in `docs/GDL_KNOWLEDGE_VERIFICATION_PLAN.zh-CN.md`.
  - Fixed the authority order used for GDL knowledge review:
    Graphisoft GDL Reference Guide, GDL Center docs, official/community support,
    OpenBrep local knowledge, then LLM inference.
- Core GDL knowledge:
  - Corrected or tightened entries for `PRISM_`, `CYLIND`, `CUTPLANE`,
    `BODY_EDGE_PGON`, `ADD/DEL`, `FOR/NEXT`, `IF/ENDIF`, `PROJECT2`,
    `HOTSPOT2`, `DEFINE`, `MATERIAL`, `GLOBALS`, `GROUP`, `SWEEP`, and
    `REVOLVE`.
  - Clarified that complex commands should be used only when the object
    geometry actually requires them.
- HSF / parameter knowledge:
  - Corrected the HSF `ParamSection` / `Parameters SectVersion="27"` knowledge
    so OpenBrep does not reason from a fake `<PARAMETERS>/<PARAMETER>` format.
  - Clarified material parameter values as Archicad attribute indices.
- Archetype knowledge:
  - Verified and updated bookshelf, cabinet, table, door, window, and profile
    object archetypes.
  - Added explicit boundaries for ordinary visual objects versus true host-aware
    Archicad door/window behavior.
- Pro direction:
  - Added a commercial Pro-layer direction plan for vertical GDL Skill packages,
    including door/window systems, enterprise drafting standards, manufacturer
    product objects, calculation/schedule objects, and cabinet/millwork.
